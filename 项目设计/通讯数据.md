# 项目通信数据格式

## 1.登陆

- 请求方法

|  url |  method   |  msg   | 
| --- | --- | --- |
|   ~/login  | POST  | post请求 |  

- 参数说明

|  字段 |  选项   |  描述   | 
|---| --- | --- |
|   account  | 必选  | 账号/手机号/身份证号 | 
|   pwd  | 必选  | 密码 |

- 示例
```json
{
    "account":"123123",
	"pwd":"123"
}
```

- 返回说明

|  字段 |  类型   |  描述   | 
|---| --- | --- |
|   code  | int  | 0: 成功返回；1：账号密码未填写；2:账号格式不正确；3：账号密码错误 | 
|   msg  | string  | 返回的消息 | 
|   user  | user结构  | 包含user类的基本信息 | 

- 示例
```json
{
   "code":"1", // 0代表操作正常，非0代表出错
   "msg":"reMsg", //返回的消息
   "user":{
         "uno":"myname",
         "name":"xx",
         "pwd":"xxx",
         "birthday":"xxx",
         "sex":"xxx",
         "idcard":"xxx",
         "phone":"xxx",
         "intro":"xxx",
         "title":"xxx",
         "role":"xxx"
    }
}
```


## 2.排班表

### 2.1 查询排班表信息（根据开始时间和结束时间）

- 请求方法

|  url |  method   |  msg   | 
| --- | --- | --- |
|   ~/pri/shift/getByDate  | GET  | GET请求 |  

- 参数说明

|  字段 |  选项   |  描述   | 
|---| --- | --- |
|   starDate  | 必选  | 开始的日期 | 
|   endDate  | 必选  | 结束日期 |

- 示例
`http://localhost:8080/pri/shift/getByDate?startDate=2018-4-29 16:00:00&endDate=2018-4-30 17:00:00`

- 返回说明

|  字段 |  类型   |  描述   | 
|---| --- | --- |
|   code  | int  | 0: 成功返回；1：日期格式不对，格式必须要是 `yyyy-mm-dd hh:mm:ss`| 
|   msg  | string  | 返回的消息 | 
|   shifts  | 排班表信息集合  | 包含多个shift类的基本信息 | 

- 示例
```json
{
    "code": "0",
    "msg": "reMsg",
    "count": "8",
    "shitfs": [
        {
            "sno": 1,
            "startDate": "2018-04-30 08:00:00.0",
            "endDate": "2018-04-30 09:00:00.0",
            "detail": "none",
            "dno": 1,
            "uno": 2,
            "maxNum": 8,
            "num": 1
        },
        {
            "sno": 2,
            "startDate": "2018-04-30 08:00:00.0",
            "endDate": "2018-04-30 09:00:00.0",
            "detail": "none",
            "dno": 3,
            "uno": 3,
            "maxNum": 8,
            "num": 1
        }
    ]
}
```

## 3 预约挂号

### 3.1 用户进行挂号

- 请求方法

|  url |  method   |  msg   | 
| --- | --- | --- |
|   ~/pri/pri/reg/addReg  | GET  | GET请求 |  

- 参数说明

|  字段 |  选项   |  描述   | 
|---| --- | --- |
|   sno  | 必选  | 排班表编号 | 
|   uno  | 必选  | 患者编号 | 

- 示例
`http://localhost:8080/pri/reg/addReg?sno=1&uno=5`

- 返回说明

|  字段 |  类型   |  描述   | 
|---| --- | --- |
|   code  | int  | 0: 预约成功；1：用户或排班编号为空;2:此时段预约已满 | 
|   msg  | string  | 返回的消息 | 

- 示例
```json
{
    "code": "0",
    "msg": "reMsg"
}
```


### 3.2 用户取消挂号

- 请求方法

|  url |  method   |  msg   | 
| --- | --- | --- |
|   ~/pri/pri/reg/delReg  | GET  | GET请求 |  

- 参数说明

|  字段 |  选项   |  描述   | 
|---| --- | --- |
|   sno  | 必选  | 排班表编号 | 
|   uno  | 必选  | 患者编号 | 

- 示例
`http://localhost:8080/pri/reg/delReg?sno=1&uno=5`

- 返回说明

|  字段 |  类型   |  描述   | 
|---| --- | --- |
|   code  | int  | 0: 取消预约成功；1:请登录；2：用户或排班编号为空;3:权限不足（试图删除别人的预约）| 
|   msg  | string  | 返回的消息 | 

- 示例
```json
{
    "code": "0",
    "msg": "reMsg"
}
```

```json
        }

```
